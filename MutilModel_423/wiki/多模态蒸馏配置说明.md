# 多模态蒸馏配置说明

## 概述

YOLOMM 和 RTDETRMM 支持可选的知识蒸馏训练，通过 `train(distill=(yaml_path, mode))` 显式开启。不传 `distill` 参数时，训练行为与普通训练完全一致。

蒸馏支持两种典型范式：
1. **两个单模态教师 -> 一个双模态学生**（RGB教师 + X教师 -> 双模态学生）
2. **一个双模态教师 -> 一个双模态小学生**（大模型教师 -> 小模型学生）

仅支持**同家族蒸馏**：YOLOMM 教师只能蒸馏 YOLOMM 学生，RTDETRMM 教师只能蒸馏 RTDETRMM 学生。

---

## 家族化 output 蒸馏设计

### 核心原则

YOLOMM 和 RTDETRMM 的 **output 蒸馏算法完全分家族实现**，不再共享同一个通用 loss：

| 家族 | output 蒸馏算法 | 实现位置 |
|------|----------------|---------|
| **YOLOMM** | 前景引导 + 分类/定位解耦蒸馏 | `nn/mm/distill/output_yolomm.py` |
| **RTDETRMM** | matching-aware / query-aware 蒸馏 | `nn/mm/distill/output_rtdetrmm.py` |

两者的区别源于检测架构的根本差异：
- YOLOMM 基于 dense anchor-free 检测头，输出是密集的逐锚点预测
- RTDETRMM 基于 DETR 的 learned query 机制，输出是稀疏的 query 集合

### YOLOMM output 蒸馏

**算法**: 前景引导 + cls/reg 解耦

1. **前景筛选**: 使用教师分类最大分数做前景选择（按尺度分别筛选，阈值 + top-k 双约束）
2. **分类蒸馏**: 在前景位置使用教师 sigmoid 输出作为 soft target，学生侧 BCEWithLogits
3. **定位蒸馏**: 在前景位置对 decoded boxes 做 L1 + GIoU 组合蒸馏
4. **预热机制**: 前 N 个 epoch 线性预热，从 0 逐步升到目标权重

训练日志中会分别记录：
- `distill_output`: output 蒸馏总损失
- `d_out_cls`: 分类蒸馏子项
- `d_out_loc`: 定位蒸馏子项

### RTDETRMM output 蒸馏

**算法**: matching-aware / query-aware

1. **有效 query 选择**: 只蒸馏教师高置信的 query（confidence 阈值 + top-k），不做全 query 蒸馏
2. **最终 decoder 层**: 仅对最终 decoder 层输出做蒸馏，不涉及多层 decoder guidance
3. **分类蒸馏**: 对选中的 query 做 soft classification distillation
4. **框蒸馏**: 对选中的 query 做 L1 + GIoU 组合蒸馏
5. **预热机制**: 与 YOLOMM 相同的线性预热策略

训练日志中会分别记录：
- `distill_output`: output 蒸馏总损失
- `d_out_cls`: 分类蒸馏子项
- `d_out_box`: 框蒸馏子项

### 教师/学生输出语义统一

教师和学生用于 output 蒸馏的输出均来自**同语义的训练态原始检测头输出**：
- 教师通过 `predict()` 获取训练态格式输出（RTDETRMM 教师在 eval 模式下会自动提取原始 5-元组）
- 学生通过 `distill_forward()` 获取训练态输出
- 两者输出空间一致，不存在语义错位

## 蒸馏 YAML 结构

蒸馏配置通过独立的 YAML 文件描述。该文件 **只描述教师集合和蒸馏映射**，不描述学生模型路径、损失函数类型或训练超参数。

**重要**: YAML 只负责声明"哪些教师参与 output 蒸馏"。具体蒸馏算法（前景引导/query-aware）由家族适配器自动决定，不由 YAML 指定。

### 顶层字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `version` | int | 否 | 配置版本号，默认 1 |
| `teachers` | list | 是 | 教师模型声明列表 |
| `mappings` | dict | 否 | 蒸馏映射分组 |

### teachers 字段

每个教师必须包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | str | 是 | 教师唯一名称（不可重复） |
| `role` | str | 是 | 运行时输入模式，仅允许 `rgb`、`x`、`dual` |
| `weights` | str | 是 | `.pt` 权重文件路径（**必须提供**） |
| `yaml` | str | 否 | 辅助 YAML 配置路径（可选，**不能替代 weights**） |

**重要**：`weights` 必须是 `.pt` 文件，不允许只有 `yaml` 没有 `weights`。

#### role 字段的运行时输入语义

**关键原则**：`role` 的含义是 **运行时给该教师喂什么输入**。它**不是**教师历史训练来源的判断字段，系统**不会**根据教师内部结构或历史训练方式反向推断其身份。

| role 值 | 运行时行为 | 输入通道 |
|---------|-----------|---------|
| `rgb` | 只取学生 batch 的 RGB 3 通道喂给教师 | 3ch |
| `x` | 只取学生 batch 的 X 模态 3 通道喂给教师 | 3ch |
| `dual` | 将完整的 RGB+X 6 通道拼接输入喂给教师 | 6ch |

**语义澄清**：

- 一个权重被声明为 `role: rgb`，系统就按 RGB 教师使用，无论该权重原始训练时使用了什么数据。
- 一个多模态训练的权重可以声明为 `role: rgb`，系统会在运行时只给它 RGB 输入。
- 一个单模态 RGB 权重也可以声明为 `role: rgb`。
- 只要教师前向能接受对应通道数的输入，声明就是合法的。
- 如果教师自身带 `mm_router`，系统会自动设置路由参数以匹配声明的 role。

**不匹配时的行为**：

- 把单模态 3 通道教师声明为 `dual` -> 通道不匹配，直接报错
- 把 6 通道 early-fusion 教师声明为 `rgb` -> 如果模型前向无法接受 3 通道输入，直接报错
- 系统不做任何"自动优雅降级"或"静默 fallback"

### mappings 字段

`mappings` 包含两个可选分组，**语义完全不同**：

#### feature 组 -- 层到层中间特征蒸馏

`feature` 组是**层索引级**蒸馏映射。每条映射指定教师-学生层对应关系，用于中间特征对齐。

**支持三种配置语法**：

**1. 原子映射**（单层对单层 -- 最直接的写法）：

```yaml
mappings:
  feature:
    - teacher: teacher_rgb
      teacher_layer: 6       # 教师模型的层索引（单个整数）
      student_layer: 17      # 学生模型的层索引（单个整数）
      tap: output            # 采集点类型，缺省即 output
```

运行时内部状态: 一条原子映射 `teacher_layer=6 -> student_layer=17`。

**2. 显式区间映射**（多层对多层，两侧长度必须一致）：

```yaml
mappings:
  feature:
    - teacher: teacher_rgb
      teacher_layer: [3, 10]    # 教师层区间 [start, end]（闭区间，共8层）
      student_layer: [13, 20]   # 学生层区间 [start, end]（闭区间，共8层）
```

运行时编译: 系统 zip 展开为 8 条原子映射:
- `T[3]->S[13]`, `T[4]->S[14]`, ..., `T[10]->S[20]`

约束:
- 两侧区间长度必须一致，否则直接报错
- `start <= end`，否则直接报错

**3. 自动展开映射**（教师区间对学生输入侧自动展开）：

```yaml
mappings:
  feature:
    - teacher: teacher_rgb
      teacher_layer: [0, 10]    # 教师层区间
      student_input: RGB        # 学生输入侧：仅允许 'RGB' | 'X' | 'DUAL'
```

运行时编译:
1. 系统在学生模型中查找 `student_input` 对应的连续 backbone 层集合
2. 要求该层集合长度与教师区间长度一致
3. 逐项 zip 展开为原子映射

约束:
- `student_input` 只允许 `RGB`、`X`、`DUAL`（解析时统一大写）
- 找不到连续 backbone 链路时直接报错，不做静默跳过
- `student_layer` 与 `student_input` 必须且只能存在一个

**原子映射数量限制**（内部常量，不暴露到 YAML）：
- 展开后总原子映射数 > 20: 发出 `LOGGER.warning` 提醒训练速度影响
- 展开后总原子映射数 > 50: 直接报错拒绝执行

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `teacher` | str | 是 | 引用的教师名称（必须在 `teachers` 中存在） |
| `teacher_layer` | int 或 [int, int] | 是 | 教师模型的层索引或闭区间 |
| `student_layer` | int 或 [int, int] | 条件必填 | 学生模型的层索引或闭区间（与 `student_input` 二选一） |
| `student_input` | str | 条件必填 | 学生输入侧标识（与 `student_layer` 二选一），仅允许 `RGB`、`X`、`DUAL` |
| `tap` | str | 否 | 采集点类型，缺省即 `output`（当前仅支持 `output`） |

#### output 组 -- 教师级输出包蒸馏声明

`output` 组是**教师参与声明**，**不写层索引**。每条声明只指定哪个教师参与输出蒸馏。系统根据模型家族自动选择对应的蒸馏算法：

- **YOLOMM 学生**: 自动使用前景引导 + cls/loc 解耦蒸馏
- **RTDETRMM 学生**: 自动使用 query-aware + cls/box 解耦蒸馏

```yaml
mappings:
  output:
    - teacher: teacher_rgb
    - teacher: teacher_x
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `teacher` | str | 是 | 引用的教师名称（必须在 `teachers` 中存在） |

**重要区别**：
- `feature` 组需要 `teacher_layer`、`student_layer`、`tap` 字段
- `output` 组只需要 `teacher` 字段，**不允许**出现 `teacher_layer`、`student_layer` 或 `tap`（出现则直接报错）

## mode 参数说明

通过 `train(distill=(yaml_path, mode))` 中的 `mode` 控制实际激活的映射分组：

| mode | 激活分组 | 说明 |
|------|----------|------|
| `output` | 仅 `mappings.output` | 只做输出级蒸馏 |
| `feature` | 仅 `mappings.feature` | 只做特征级蒸馏 |
| `both` | 两组都激活 | 同时做输出级和特征级蒸馏 |

### 忽略规则

- YAML 中未被当前 `mode` 启用的 mapping 分组 **在训练时会被完全忽略**
- 忽略不等于执行：未启用的分组不会注册采集点、不会执行前向、不会参与 loss 计算
- 两个分组都可以写在同一个 YAML 中，由 `mode` 决定实际激活哪些

## 特征蒸馏算法

### 前景引导的检测友好特征蒸馏

特征蒸馏采用**前景引导 + 多项损失组合**的检测友好设计，而不是简单的全图 MSE。

**1. 前景软掩码生成**：
- **YOLOMM**：从教师输出中选择高置信框（`score >= 0.05` + `topk=100`），投影到特征图空间
- **RTDETRMM**：从教师有效 query 中选择高置信框，投影到特征图空间
- 每个框的掩码权重为 `score / sqrt(area + eps)`，多框重叠时累加并裁剪到 `[0, 1]`

**2. 最终损失公式**：

对每个 teacher-student 特征映射，计算：

```
L_map = w_map * (
    lambda_fg  * L_fg    # 前景区域归一化 SmoothL1（主损失）
  + lambda_bg  * L_bg    # 背景区域归一化 SmoothL1（弱约束）
  + lambda_cwd * L_cwd   # 前景区域通道分布蒸馏
  + lambda_ctx * L_ctx   # 前景区域全局语义向量 cosine loss
)
```

其中：
- **L_fg**：前景区域上的归一化特征 SmoothL1 损失。对归一化后的 feature 计算逐点 SmoothL1，再按 fg_mask 做加权平均。这是主要的蒸馏信号。
- **L_bg**：背景区域上的归一化特征 SmoothL1 损失。使用 `bg_mask = 1 - fg_mask`，权重较小（默认 0.25），作为弱约束防止背景漂移。
- **L_cwd**：前景区域上的 Channel-Wise Distribution KL 散度。对每个通道在前景空间维度展平，计算 `softmax(logits / tau)` 后的 KL 散度，帮助学生学习教师的通道激活模式。
- **L_ctx**：前景区域全局池化语义向量的 cosine 距离。对 fg_mask 加权池化出 `B x C` 语义向量，再计算 `1 - cosine_similarity`，确保高层语义一致性。
- **w_map**：分层权重，按层深度递增，越接近检测语义末端的层权重越大。

**3. 默认超参**（内部常量，不暴露到 YAML）：

```python
FEATURE_DISTILL_WEIGHT = 1.0   # feature 蒸馏总权重
FEATURE_FG_WEIGHT      = 1.0   # 前景损失权重
FEATURE_BG_WEIGHT      = 0.25  # 背景损失权重
FEATURE_CWD_WEIGHT     = 0.5   # 通道分布权重
FEATURE_CTX_WEIGHT     = 0.25  # 上下文语义权重
FEATURE_CWD_TAU        = 1.0   # CWD 温度
FEATURE_WARMUP_RATIO   = 0.15  # 预热比例
```

**4. 训练日志**：

特征蒸馏会记录以下详细指标：
- `distill_feature`：特征蒸馏总损失
- `d_feat_fg`：前景损失子项
- `d_feat_bg`：背景损失子项
- `d_feat_cwd`：通道分布损失子项
- `d_feat_ctx`：上下文语义损失子项

### 特征对齐规则

| 情况 | 处理方式 |
|------|----------|
| 空间尺寸不一致 | 自动通过双线性插值对齐到学生特征尺寸 |
| 通道数不一致 | **直接报错**，不做自动对齐 |

## 多教师损失聚合策略

双单模态教师（`role=rgb` + `role=x`）时：

- 输出蒸馏和特征蒸馏都采用 **分别计算、再加权求和**
- 不会先融合两个教师的结果
- 每个教师的损失项等权平均

单教师（`role=dual`）时：

- 只生成一条教师损失项

## 使用示例

### YOLOMM 蒸馏训练

```python
from ultralytics import YOLOMM

# 创建学生模型
student = YOLOMM('yolo11n-mm-mid.yaml')

# 开启蒸馏训练
student.train(
    data='multimodal-dataset.yaml',
    distill=('path/to/distill_config.yaml', 'both'),  # 同时启用输出和特征蒸馏
    epochs=100,
    batch=16,
)
```

### RTDETRMM 蒸馏训练

```python
from ultralytics import RTDETRMM

student = RTDETRMM('rtdetr-r18-mm-mid.yaml')

student.train(
    data='multimodal-dataset.yaml',
    distill=('path/to/distill_config.yaml', 'output'),  # 仅输出蒸馏
    epochs=100,
    batch=4,
)
```
